NPM
  • Package Manager for Javascript
  • Can manage dependencies and versions
  • Ships with node.js
  • Makes it easy for JavaScript developers to share and reuse code
  • Makes it easy to update the code that you're sharing

Development tools

NPM - Package.json
  • Name: The unique name of your package. Required
  • Scripts: A object which exposes addition npm commands.
  • Main: The main entry point to your package. When calling require('{module name}')
  • Dependencies: An object which contain the dependencies of your package
  • devDependencies: These are the dependencies that are only intended for development and testing of your module
  • preferGlobal: A property that indicates that this page prefers to be installed globally using npm install -g {module-name}

Development tools

NPM - package.json dependencies
  • 1.2.1Must match version exactly
  • >1.2.1Must be greater than version
  • >=1.2.1Must be greater or equal than version
  • <1.2.1Must be lower than version
  • <=1.2.1Must be lower or equal than version
  • ^1.2.1Must be a compatible with version

Bower
  • Front-end dependencies Manager
  • Bower, unlike npm, can have multiple files (e.g. .js, .css, .html, .png, .ttf) which are considered the main file(s).
  • Bower semantically considers these main files, when packaged together, a component.

Jasmine
  • Framework for testing javascript code
  • Does not depend on any other javascript framework
  • Does not require a DOM

Development tools

Jasmine - tests

Development tools

Jasmine - tests
  • A test suite begins with a call to the global Jasmine function describe
  • 2 parameters: a string and a function
    • The string is a name or title for a spec suite
    • The function is a block of code that implements the suite.

Development tools

Jasmine - Specs

Development tools

Jasmine - Specs
  • Specs are defined by calling the global Jasmine function it
  • The string is the title of the spec
  • The function is the spec, or test
  • A spec contains one or more expectations that test the state of the code

Development tools

Jasmine - it's just function
  • Describe and it blocks are functions
  • They can contain any executable code necessary to implement the test
  • JavaScript scoping rules apply: variables declared in a describe are available to any it block inside the suite.

Gulp - how to use it ?
Create a task
gulp.task('somename', function() {
  // Do stuff
});
Create a task with dependencies (will be executed and completed before the task
gulp.task('mytask', ['array', 'of', 'task', 'names'], function() {
  // Do stuff
});
Emits files matching
gulp.src('client/templates/*.jade')
  .pipe(jade())
  .pipe(minify())
  .pipe(gulp.dest('build/minified_templates'));
Write files
gulp.src('./client/templates/*.jade')
  .pipe(jade())
  .pipe(gulp.dest('./build/templates'))
  .pipe(minify())
  .pipe(gulp.dest('./build/minified_templates'));

Gulp vs Grunt
  • Grunt and Gulp do exactly the same thing
  • Gulp has been developed to solve issues you may have encountered with Grunt
  • Grunt requires plug-ins for basic functionality such as file watching (built-in for Gulp)
  • Grunt uses JSON-like data configuration files (plain JS for Gulp)
  • lot of add-ons and plugins for both
