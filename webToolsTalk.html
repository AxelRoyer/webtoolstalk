During the development phase - NPM
  • Package Manager for Javascript
  • Can manage dependencies and versions
  • ships with node.js
  • Makes it easy for JavaScript developers to share and reuse code
  • Makes it easy to update the code that you're sharing

Development tools

During the development phase - NPM

package.json
  • Name: The unique name of your package. Required
  • Scripts: A object which exposes addition npm commands. The object assumes that the key is the npm command and the value is the script path
  • Main: The main entry point to your package. When calling require('{module name}')
  • Dependencies: An object which contain the dependencies of your package
  • devDependencies: These are the dependencies that are only intended for development and testing of your module
  • preferGlobal: A property that indicates that this page prefers to be installed globally using npm install -g {module-name}

Jasmine
  • BDD framework for testing javascript code
  • does not depend on any other javascript framework
  • does not require a DOM

Development tools

Jasmine - tests
  • A test suite begins with a call to the global Jasmine function describe
  • 2 parameters: a string and a function
  • The string is a name or title for a spec suite
  • The function is a block of code that implements the suite.

Development tools

Jasmine - tests

Development tools

Jasmine - Specs
  • pecs are defined by calling the global Jasmine function it
  • The string is the title of the spec
  • The function is the spec, or test
  • A spec contains one or more expectations that test the state of the code

Development tools

Jasmine - Specs

Development tools

Jasmine - it's just function
  • Describe and it blocks are functions
  • They can contain any executable code necessary to implement the test
  • JavaScript scoping rules apply: variables declared in a describe are available to any it block inside the suite.

Jasmine - setup and teardown code
  • BeforeEach function is called once before each spec in the describe is run
  • AfterEach function is called once after each spec
  • beforeAll function is called only once before all the specs in describe are run
  • afterAll function is called after all specs finish

Mocha, Chai, Sinon vs Jasmine

Debug - Profile panel
  • Lets you profile the execution time and memory usage of a web app or page
  • The Heap profiler shows memory distribution by your page's JavaScript objects and related DOM nodes
